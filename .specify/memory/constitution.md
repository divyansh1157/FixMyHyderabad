# Project Constitution: FixMyHyderabad

> Version: 1.0.0
> Ratified: 2026-05-31
> Last Amended: 2026-05-31

---

## 1. Core Principles

### Principle 1: Community-Driven Verification
**Rules:**
- All reported issues MUST be confirmable by other citizens.
- The platform MUST prioritize issues with higher community confirmation counts.
- Duplicate confirmations from the same session MUST be blocked.

**Rationale:**
- Crowd verification ensures that GHMC resources are focused on genuine, high-impact issues verified by the community.

### Principle 2: Real-Time Transparency
**Rules:**
- The issue feed MUST update in real-time without requiring a page refresh.
- Issue status (Active, In Review, Resolved) MUST be clearly visible and accurate.
- Submission timestamps MUST be transparent to show issue age.

**Rationale:**
- Real-time updates build trust between citizens and authorities by providing immediate visibility into civic problems.

### Principle 3: Privacy-First Reporting
**Rules:**
- No personal user accounts or login SHOULD be required for reporting or confirming.
- Browser session IDs MUST be used anonymously for duplicate prevention.
- User location data MUST only be used for the specific purpose of tagging a report.

**Rationale:**
- Lowering the barrier to entry and protecting citizen privacy encourages wider participation in civic pulse reporting.

---

## 2. Governance

### Amendment Procedure
Amendments to this constitution require a proposal and agreement from the project maintainers. Changes must be documented in the version history.

### Versioning Policy
- MAJOR: Backward incompatible governance or principle removals.
- MINOR: New principles or materially expanded guidance.
- PATCH: Clarifications, wording, or non-semantic refinements.

### Compliance Review
The platform's alignment with these principles will be reviewed periodically during development sprints to ensure technical decisions support core civic goals.
